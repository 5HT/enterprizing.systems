Distributed, fault tolerant, network split resistant, performance, transactional business processing system.

Ontological Model

We define data model from the ground up to avoid problems in future. In top-level metainformation you are able to define rules for storing, displaying and treating the tuples.

Typed Schema

Schema definition for various database backends: Mnesia, Riak and KAI; is generated from top-level metainformation and strictly defines JSON/XML and other format entities.

Generated Forms

We automatically generates the visual forms for data assessment based on ontological model. These forms are fulfilled with native data for KVS storage.

Workflow Engine

Define your business logic compatible with workflow vendors and understandable by system analytics.

Decision Trees

Data Pattern matching builds effective decision trees from business rules and can compete with Drools. See RETE algorithm.

Relational Export

All data is able to be exported to SQL warehouses for offline analysis and complex audit reporting.

Unified Messaging

SpawnProc is Message Bus and Message Format agnostic. We provide libraries for AMQP, MQTT and other pub/sub providers. Our messaging relay N2O supports number of transports: JSON, Binary, XML, TXT; and could be plugged directly to WebSockets front-end.

Distributed XA

SpawnProc ships with distributed transaction coordinator that turns any component to be transaction aware.
